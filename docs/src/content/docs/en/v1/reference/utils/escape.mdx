## quoteStr

The `quoteStr` function quotes a string. For convenience, it also quotes empty strings.

### Import

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function quoteStr(str: string): string;
```

Quotes a string with either `"` or `'`.

#### Arguments

##### `str`

The string to quote.

#### Return Value

Returns the quoted string. The quote character depends on whether the given string contains single quotes (`'`):

- When the given string is empty
  - Returns `''`.
- When the given string does **not** contain single quotes
  - Quotes with single quotes.
- When the given string does **contain** single quotes
  - Quotes with double quotes.
- When the given string contains both single and double quotes
  - Quotes with double quotes.
  - Escapes double quotes with backslashes (`\`).

#### Example

The following example shows how to quote a string that **does not** contain single quotes:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(``));
console.log(quoteStr(`a`));
console.log(quoteStr(`𩸽`));
console.log(quoteStr(`👍🏽`));
console.log(quoteStr(`パ`));
```

Output:

```js
''
'a'
'𩸽'
'👍🏽'
'パ'
```

The following example shows how to quote a string that **does** contain single quotes:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(`cat's`));
console.log(quoteStr(`𩸽's`));
console.log(quoteStr(`'👍🏽`));
console.log(quoteStr(`'パ`));
```

Output:

```js
"cat's"
"𩸽's"
"'👍🏽"
"'パ"
```

The following example shows how to quote a string that contains both single and double quotes, escaping when necessary:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(`cat's "toy"`));
console.log(quoteStr(`'\\"\\`));
console.log(quoteStr(`𩸽's "feed"`));
console.log(quoteStr(`'👍🏽"`));
console.log(quoteStr(`'パ"`));
```

Output:

```js
"cat's \"toy\""
"'\\\"\\"
"𩸽's \"feed\""
"'👍🏽\""
"'パ\""
```

## escapeKey

The `escapeKey` function is similar to [`quoteStr`](/en/reference/utils/quote-str/), but is used for object keys. For convenience, it also quotes empty strings.

### Import

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeKey(key: string): string;
```

#### Arguments

##### `key`

The object key.

#### Return Value

Returns the escaped string, quoting with double quotes if necessary:

- When the given string is empty
  - Returns `""`.
- When the given string consists only of alphanumeric characters and underscores
  - Returns the string as is.
- When the given string contains anything other than alphanumeric characters and underscores
  - Quotes with double quotes.
- When the given string contains backslashes (`\`)
  - Escapes them.
- When the given string contains double quotes
  - Escapes them.

#### Example

The following example shows how to escape a string that consists only of alphanumeric characters and underscores, resulting in the string as is:

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";

console.log(escapeKey(`123`));
console.log(escapeKey(`foo_bar`));
```

Output:

```js
123
foo_bar
```

The following example shows how to escape a string, quoting with double quotes and escaping when necessary:

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";

console.log(escapeKey(``));
console.log(escapeKey(`foo-bar`));
console.log(escapeKey(`'\\"\\`));
console.log(escapeKey(`𩸽's "feed"`));
console.log(escapeKey(`'👍🏽"`));
console.log(escapeKey(`'パ"`));
```

Output:

```js
""
"foo-bar"
"'\\\"\\"
"𩸽's \"feed\""
"'👍🏽\""
"'パ\""
```

## escapeRid

The `escapeRid` function is similar to [`quoteStr`](/en/reference/utils/quote-str/), but is used for record IDs. For convenience, it also quotes empty strings.

### Import

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeRid(rid: string): string;
```

#### Arguments

##### `rid`

The ID portion of the record ID.

#### Return Value

Returns the escaped string, quoting with brackets (`⟨⟩`) if necessary:

- When the given string is empty
  - Returns `⟨⟩`.
- When the given string does not start with a number and consists only of alphanumeric characters and underscores
  - Returns the string as is.
- When the given string contains brackets (`⟩`)
  - Escapes them.
- When the given string starts with a number or contains anything other than alphanumeric characters and underscores
  - Quotes with brackets (`⟨⟩`).

#### Example

The following example shows how to escape a string that does not start with a number and consists only of alphanumeric characters and underscores, resulting in the string as is:

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";

console.log(escapeRid(`foo_123`));
```

Output:

```js
foo_123
```

The following example shows how to escape a string, quoting with brackets (`⟨⟩`) and escaping when necessary:

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";

console.log(escapeRid(``));
console.log(escapeRid(`123`));
console.log(escapeRid(`foo-bar`));
console.log(escapeRid(`𩸽's ⟨feed⟩`));
console.log(escapeRid(`'👍🏽⟩`));
console.log(escapeRid(`'パ⟩`));
```

Output:

```js
⟨⟩
⟨123⟩
⟨foo-bar⟩
⟨𩸽's ⟨feed\⟩⟩
⟨'👍🏽\⟩⟩
⟨'パ\⟩⟩
```

## escapeIdent

The `escapeIdent` function is similar to [`quoteStr`](/en/reference/utils/quote-str/), but is used for namespaces, databases, tables, and field names. For convenience, it also quotes empty strings.

### Import

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeIdent(ident: string): string;
```

#### Arguments

##### `ident`

The identifier.

#### Return Value

Returns the escaped string, quoting with backticks (<code>`</code>) if necessary:

- When the given string is empty
  - Returns <code>``</code>.{/* TODO(tai-kun): " になる問題解決する */}
- When the given string does not start with a number and consists only of alphanumeric characters and underscores
  - Returns the string as is.
- When the given string contains backticks (<code>`</code>)
  - Escapes them.
- When the given string starts with a number or contains anything other than alphanumeric characters and underscores
  - Quotes with backticks (<code>`</code>).

#### Example

The following example shows how to escape a string that does not start with a number and consists only of alphanumeric characters and underscores, resulting in the string as is:

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";

console.log(escapeIdent(`foo_123`));
```

Output:

```js
foo_123
```

The following example shows how to escape a string, quoting with backticks (<code>`</code>) and escaping when necessary:

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";

console.log(escapeIdent(""));
console.log(escapeIdent("123"));
console.log(escapeIdent("foo-bar"));
console.log(escapeIdent("𩸽's `feed`"));
console.log(escapeIdent("'👍🏽`"));
console.log(escapeIdent("'パ`"));
```

Output:

```js
``
`123`
`foo-bar`
`𩸽's \`feed\``
`'👍🏽\``
`'パ\``
```

## Notes

### String Length

The strings used in the previous examples contain characters that have different string lengths as seen by JavaScript and humans:

| Type | Display | `.length` | `Uint8Array` |
| --- | --- | --- | --- |
| ASCII | a | 1 | `[ 0x61 ]` |
| Surrogate Pair | 𩸽 | 2 | `[ 0xF0, 0xA9, 0xB8, 0xBD ]` |
| Variation Selector | 👍🏽 | 4 | `[ 0xF0, 0x9F, 0x91, 0x8D, 0xF0, 0x9F, 0x8F, 0xBD ]` |
| Combining Character | パ | 2 | `[ 0xE3, 0x83, 0x8F, 0xE3, 0x82, 0x9A ]` |

When the `.length` value is counterintuitive, it is necessary to carefully consider the operations on the string.
