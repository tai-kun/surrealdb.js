The `Datetime` class represents a date and time, corresponding to the SurrealQL [`datetime`](https://surrealdb.com/docs/surrealql/datamodel/datetimes) type.

### Import

```ts
import { Datetime } from "@tai-kun/surrealdb/decodeonly-datatypes";
import { Datetime } from "@tai-kun/surrealdb/encodable-datatypes";
import { Datetime } from "@tai-kun/surrealdb/standard-datatypes";
```

### `.constructor()`

Creates a new `Datetime` object.

```ts
new Datetime(source: [number, number] | string); // decode-only, encodable
new Datetime(
  source?:
    | [number, number]
    | number
    | bigint
    | Date
    | Datetime
    | undefined,
); // standard
new Datetime(
  year: number,
  monthIndex: number,
  date?: number | undefined,
  hours?: number | undefined,
  minutes?: number | undefined,
  seconds?: number | undefined,
  milliseconds?: number | undefined,
  microseconds?: number | undefined,
  nanoseconds?: number | undefined,
); // standard
```

#### Arguments

##### `source`

A pair of seconds and nanoseconds, or a string. For the standard preset, it also supports `Date` objects and `bigint` inputs.

##### `year`, `monthIndex`, ...

Same as the arguments for the `Date` class. However, it also supports `nanoseconds` input. Providing a `number` after a `undefined` argument is not allowed. This is roughly the same specification as the `Date` class.

#### Returns

When called via `new`, `Datetime` returns its instance. If the preset is standard, it is also an instance of `Date`.

### `.seconds` <Badge variant="instance" /><Badge variant="property" /><Badge variant="decodeonly" />

The seconds part of the nanosecond-precision timestamp. It is read-only for presets below standard. The value is an integer between -2<sup>53</sup>-1 and 2<sup>53</sup>-1, or `NaN`.

```ts
.seconds: number;
```

### `.nanoseconds` <Badge variant="instance" /><Badge variant="property" /><Badge variant="decodeonly" />

The nanoseconds part of the nanosecond-precision timestamp. It is read-only for presets below standard. The value is an integer between 0 and 999999999, or `NaN`.

```ts
.nanoseconds: number;
```

### `.valueOf()` <Badge variant="instance" /><Badge variant="method" /><Badge variant="encodable" />

Retrieves the millisecond-precision timestamp.

```ts
.valueOf(): number;
```

#### Arguments

None.

#### Returns

Returns the millisecond-precision timestamp.

#### Example

```ts
import { Datetime } from "@tai-kun/surrealdb/encodable-datatypes";

const date = new Datetime("2024-06-01T12:34:56.780123456Z");
console.log(date.valueOf());
//-> 1717245296780
```

### `.toISOString()` <Badge variant="instance" /><Badge variant="method" /><Badge variant="encodable" />

Retrieves the string representation of the nanosecond-precision timestamp in ISO format. Similar to the `.toISOString()` method of the `Date` object, but it serializes to the ninth decimal place because of the nanosecond precision.

```ts
.toISOString(): string;
```

#### Arguments

None.

#### Returns

Returns the string representation of the nanosecond-precision timestamp in ISO format.

#### Example

```ts
import { Datetime } from "@tai-kun/surrealdb/encodable-datatypes";

const date = new Datetime([1717245296, 780123456]);
console.log(date.toISOString());
//-> 2024-06-01T12:34:56.780123456Z
```

Converts to a runtime built-in `Date` object. Note that it will be millisecond-precision.

```ts

```

#### Arguments

None.

#### Returns

Returns a runtime built-in `Date` object.

### `.toSurql()` <Badge variant="instance" /><Badge variant="method" /><Badge variant="encodable" />

Converts a `Datetime` object to a string that can be embedded in SurrealQL. Similar to `.toISOString()`, but it adds a `d` prefix to tell the query parser that the string is a UUID.

```ts
.toSurql(): string;
```

#### Arguments

None.

#### Returns

Returns the ISO-formatted date and time with a `d` prefix.

#### Example

```ts
import { Datetime } from "@tai-kun/surrealdb/encodable-datatypes";

const date = new Datetime([1717245296, 780123456]);
console.log(date.toSurql());
//-> d'2024-06-01T12:34:56.780123456Z'
```

### `.getMicroseconds()` <Badge variant="instance" /><Badge variant="method" /><Badge variant="standard" />

Retrieves the microseconds of the date using local time.

```ts

```

#### Arguments

None.

#### Returns

Returns the microseconds.

### `.setMicroseconds()` <Badge variant="instance" /><Badge variant="method" /><Badge variant="standard" />

Sets the microseconds.

`.setUTCMicroseconds()` behaves the same way as this method.

```ts
.setMicroseconds(us: number): number;
```

#### Arguments

##### `us`

The microseconds to set. The value will be converted to an integer. Note that if the value is not between 0 and 999, the entire timestamp will be affected.

#### Returns

Returns the millisecond-precision timestamp.

#### Example

```ts
import { Datetime } from "@tai-kun/surrealdb/standard-datatypes";

const date = new Datetime([1717245296, 780123456]);
console.log(date.toISOString());
//-> 2024-06-01T12:34:56.780123456Z

const time = date.setMicroseconds(1_000);
console.log(date.toISOString());
//-> 2024-06-01T12:34:56.781000456Z
console.log(time);
//-> 1717245296781
```

### Further Development

#### Generic Determination of `Datetime` Objects

To determine if a value is a `Datetime` object without relying on the preset, it is recommended to use the `isDatetime` function. This function does not use `instanceof`, so it is helpful when the preset of the value to be checked is unknown.

```ts
import { isDatetime } from "@tai-kun/surrealdb";
```

#### Dates Before UNIX Epoch

The `datetime` type in SurrealDB can hold nanosecond precision. Therefore, care must be taken when handling them in JavaScript. If you pass a SurrealDB-serialized `datetime`, such as `"1969-12-31T23:59:59.999999999Z"`, to JavaScript's `Date.parse`, it will be 0 milliseconds in WebKit, but -1 millisecond in other browsers (Node.js, Deno, Bun, Chromium, Firefox).

WebKit:

```ts
const date = new Date("1969-12-31T23:59:59.999999999Z");
console.log(date.getTime());
//-> 0
```

Node.js, Deno, Bun, Chromium, Firefox:

```ts
const date = new Date("1969-12-31T23:59:59.999999999Z");
console.log(date.getTime());
//-> -1
```
