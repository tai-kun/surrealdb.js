## quoteStr

`quoteStr` is a function that quotes a string. For convenience, it also quotes an empty string.

### Import

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function quoteStr(str: string): string;
```

It quotes a string with either a double quote (`"`) or a single quote (`'`).

#### Arguments

##### `str`

The string to be quoted.

#### Return Value

It returns the quoted string. The quote used for quoting depends on whether the given string contains single quotes (`'`):

- If the given string is empty
  - Returns `''`.
- If the given string **does not** contain a single quote
  - Quotes with a single quote.
- If the given string **does** contain a single quote
  - Quotes with a double quote.
- If the given string contains both single quotes and double quotes
  - Quotes with a double quote.
  - Escapes double quotes with a backslash (`\`)

#### Examples

The following example quotes a string that **does not** contain a single quote with a single quote:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(``));
console.log(quoteStr(`a`));
console.log(quoteStr(`𩸽`));
console.log(quoteStr(`👍🏽`));
console.log(quoteStr(`パ`));
```

Output:

```js
''
'a'
'𩸽'
'👍🏽'
'パ'
```

The following example quotes a string that **does** contain a single quote with a double quote:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(`cat's`));
console.log(quoteStr(`𩸽's`));
console.log(quoteStr(`'👍🏽`));
console.log(quoteStr(`'パ`));
```

Output:

```js
"cat's"
"𩸽's"
"'👍🏽"
"'パ"
```

The following example quotes a string that contains both single quotes and double quotes with a double quote and escapes them if needed:

```ts
import { quoteStr } from "@tai-kun/surrealdb/utils";

console.log(quoteStr(`cat's "toy"`));
console.log(quoteStr(`'\\"\\`));
console.log(quoteStr(`𩸽's "feed"`));
console.log(quoteStr(`'👍🏽"`));
console.log(quoteStr(`'パ"`));
```

Output:

```js
"cat's \"toy\""
"'\\\"\\"
"𩸽's \"feed\""
"'👍🏽\""
"'パ\""
```

## escapeKey

`escapeKey` is similar to [`quoteStr`](/en/reference/utils/quote-str/), but it is a function for object keys. For convenience, it also quotes an empty string.

### Import

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeKey(key: string): string;
```

#### Arguments

##### `key`

An object key.

#### Return Value

It returns an escaped string. It quotes it with double quotes if needed:

- If the given string is empty
  - Returns `""`.
- If the given string is composed of only alphanumeric characters and underscores
  - Returns the string as is.
- If the given string contains characters other than alphanumeric characters and underscores
  - Quotes with double quotes.
- If the given string contains a backslash (`\`)
  - Escapes it.
- If the given string contains a double quote
  - Escapes it.

#### Examples

The following example returns the string as is because it is composed of only alphanumeric characters and underscores:

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";

console.log(escapeKey(`123`));
console.log(escapeKey(`foo_bar`));
```

Output:

```js
123
foo_bar
```

The following example quotes the string with double quotes and escapes if needed:

```ts
import { escapeKey } from "@tai-kun/surrealdb/utils";

console.log(escapeKey(``));
console.log(escapeKey(`foo-bar`));
console.log(escapeKey(`'\\"\\`));
console.log(escapeKey(`𩸽's "feed"`));
console.log(escapeKey(`'👍🏽"`));
console.log(escapeKey(`'パ"`));
```

Output:

```js
""
"foo-bar"
"'\\"\"
"𩸽's \"feed\""
"'👍🏽\""
"'パ\""
```

## escapeRid

`escapeRid` is similar to [`quoteStr`](/en/reference/utils/quote-str/), but it is a function for record IDs. For convenience, it also quotes an empty string.

### Import

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeRid(rid: string): string;
```

#### Arguments

##### `rid`

The ID part of the record ID.

#### Return Value

It returns an escaped string. It quotes it with brackets (`⟨⟩`) if needed:

- If the given string is empty
  - Returns `⟨⟩`.
- If the given string does not start with a digit and is composed of only alphanumeric characters and underscores
  - Returns the string as is.
- If the given string contains a bracket (`⟩`)
  - Escapes it.
- If the given string is composed of only digits, or contains characters other than alphanumeric characters and underscores
  - Quotes with brackets (`⟨⟩`).

#### Examples

The following example returns the string as is because it does not start with a digit and is composed of only alphanumeric characters and underscores

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";

console.log(escapeRid(`foo_123`));
```

Output:

```js
foo_123
```

The following example quotes the string with brackets (`⟨⟩`) and escapes if needed:

```ts
import { escapeRid } from "@tai-kun/surrealdb/utils";

console.log(escapeRid(``));
console.log(escapeRid(`123`));
console.log(escapeRid(`foo-bar`));
console.log(escapeRid(`𩸽's ⟨feed⟩`));
console.log(escapeRid(`'👍🏽⟩`));
console.log(escapeRid(`'パ⟩`));
```

Output:

```js
⟨⟩
⟨123⟩
⟨foo-bar⟩
⟨𩸽's ⟨feed\⟩⟩
⟨'👍🏽\⟩⟩
⟨'パ\⟩⟩
```

## escapeIdent

`escapeIdent` is similar to [`quoteStr`](/en/reference/utils/quote-str/), but it is a function for namespaces, databases, tables, and field names. For convenience, it also quotes an empty string.

### Import

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";
```

### Usage

```ts
function escapeIdent(ident: string): string;
```

#### Arguments

##### `ident`

An identifier.

#### Return Value

It returns an escaped string. It quotes it with backticks (<code>`</code>) if needed:

- If the given string is empty
  - Returns <code>``</code>.{/* TODO(tai-kun): " になる問題解決する */}
- If the given string does not start with a digit and is composed of only alphanumeric characters and underscores
  - Returns the string as is.
- If the given string contains a backtick (<code>`</code>)
  - Escapes it.
- If the given string starts with a digit, or contains characters other than alphanumeric characters and underscores
  - Quotes with backticks (<code>`</code>).

#### Examples

The following example returns the string as is because it does not start with a digit and is composed of only alphanumeric characters and underscores

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";

console.log(escapeIdent(`foo_123`));
```

Output:

```js
foo_123
```

The following example quotes the string with backticks (<code>`</code>) and escapes if needed:

```ts
import { escapeIdent } from "@tai-kun/surrealdb/utils";

console.log(escapeIdent(""));
console.log(escapeIdent("123"));
console.log(escapeIdent("foo-bar"));
console.log(escapeIdent("𩸽's `feed`"));
console.log(escapeIdent("'👍🏽`"));
console.log(escapeIdent("'パ`"));
```

Output:

```js
``
`123`
`foo-bar`
`𩸽's \`feed\``
`'👍🏽\``
`'パ\``
```

### String Length

The strings used in the previous examples contain characters that have different lengths when viewed from JavaScript and humans:

| Type | Display | `.length` | `Uint8Array` |
| --- | --- | --- | --- |
| ASCII | a | 1 | `[ 0x61 ]` |
| Surrogate Pair | 𩸽 | 2 | `[ 0xF0, 0xA9, 0xB8, 0xBD ]` |
| Variation Selector | 👍🏽 | 4 | `[ 0xF0, 0x9F, 0x91, 0x8D, 0xF0, 0x9F, 0x8F, 0xBD ]` |
| Combining Character | パ | 2 | `[ 0xE3, 0x83, 0x8F, 0xE3, 0x82, 0x9A ]` |

You need to be careful when operating on strings if the value of `.length` is not intuitive.
