title: StatefulPromise

import Badge from "~/components/en/Badge.astro";

`StatefulPromise` is a class that implements the `PromiseLike` interface, and doesn't throw errors when it's rejected in an unhandled state. 

### Import

```ts
import { StatefulPromise } from "@tai-kun/surrealdb/utils";
```

### `.constructor()`

```ts
new StatefulPromise<T>(executor: StatefulPromiseExecutor<T>);
```

#### Arguments

##### `executor`

It plays the same role as the [`executor`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise/Promise#executor) in the `Promise()` constructor for the `StatefulPromise`.

#### Return Value

When called with `new`, `StatefulPromise` returns its instance.

#### Examples

##### Resolved

In the following example, we wait for the `StatefulPromise` to be resolved and receive the result:

```ts
import { StatefulPromise } from "@tai-kun/surrealdb/utils";

const promise = new StatefulPromise(resolve => {
  setTimeout(() => resolve("test"), 0);
});

const result = await promise;

console.log(result); // "test"
```

##### Rejected

In the following example, we wait for the `StatefulPromise` to be rejected and expect it to throw an error when handled:

```ts
const promise = new StatefulPromise((_, reject) => {
  setTimeout(() => reject("test"), 0);
});

while (promise.state === "pending") {
  await new Promise(r => setTimeout(r, 50));
}

try {
  await promise;
} catch (e) {
  console.log(e); // "test"
}
```

We can see that while a normal `Promise` would throw an `unhandledrejection` if it's rejected in an unhandled state (meaning `reject` is called), `StatefulPromise` doesn't throw an error until it's explicitly handled.

### `.state` <Badge variant="instance" /><Badge variant="property" /><Badge variant="readonly" />

Represents the current state of the `StatefulPromise`. There are three possible states:

- `pending`: Indicates it's pending. It has not been resolved or rejected yet.
- `fulfilled`: Indicates it's resolved.
- `rejected`: Indicates it's rejected.

### `.then()` <Badge variant="instance" /><Badge variant="method" />

`.then` takes callback functions for both success and failure cases of the `StatefulPromise` and returns a `StatefulPromise` object.

```ts
then<R1, R2>(
  onFulfilled?: ((value: any) => R1 | PromiseLike<R1>) | undefined | null,
  onRejected?: ((reason: unknown) => R2 | PromiseLike<R2>) | undefined | null,
): StatefulPromise<R1 | R2>;
```

#### Arguments

The same as [`Promise.prototype.then`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise/then#onfulfilled).

#### Return Value

It returns a `StatefulPromise` object.

### `.resolve()` <Badge variant="static" /><Badge variant="method" />

`.resolve` returns a `StatefulPromise` resolved with the given value.

```ts
resolve<T>(value?: T | PromiseLike<T>): StatefulPromise<Awaited<T>>;
```

#### Arguments

##### `value`

The value to resolve. The default value is `undefined`.

#### Return Value

It returns a `StatefulPromise` object resolved with the given `value`. If the value is a `StatefulPromise` object, it returns that value.

### `.reject()` <Badge variant="static" /><Badge variant="method" />

`.reject` returns a `StatefulPromise` object rejected with the given reason.

```ts
reject<T>(reason?: unknown): StatefulPromise<T>;
```

#### Arguments

##### `reason`

The reason to reject. The default value is `undefined`.

#### Return Value

It returns a `StatefulPromise` rejected with the given `reason`.

### `.withResolvers()` <Badge variant="static" /><Badge variant="method" />

`.withResolvers` is a function that returns a new `StatefulPromise` object and an object containing functions to resolve or reject it.

```ts
withResolvers<T>(): {
  promise: StatefulPromise<T>;
  resolve: (value: T | PromiseLike<T>) => void;
  reject: (reason: unknown) => void;
};
```

#### Arguments

None.

#### Return Value

It returns a plain object with the following properties.

##### `promise`

A new `StatefulPromise` object.

##### `resolve`

A function to resolve the `promise`.

##### `reject`

A function to reject the `promise`.

### `.try()` <Badge variant="static" /><Badge variant="method" />

`.try` is a function that wraps a function in a `StatefulPromise`.

```ts
try<T, A extends readonly unknown[]>(
  func: (...args: A) => T | PromiseLike<T>,
  ...args: A
): StatefulPromise<T>;
```

#### Arguments

##### `func`

The function to be called. This function can return any value except a `StatefulPromise`. It can also throw errors synchronously inside the function.

#### Return Value

- If `func` returns a value synchronously, it returns a `StatefulPromise` in a resolved state.
- If `func` throws an error synchronously, it returns a `StatefulPromise` in a rejected state.
- If `func` resolves or rejects asynchronously, it returns a `StatefulPromise` in a pending state.

#### Examples

The following example shows that an error thrown synchronously is caught by the `StatefulPromise`.

```ts
import { StatefulPromise } from "@tai-kun/surrealdb/utils";

const promise = StatefulPromise.try(() => {
  throw "test";
});

await promise.then(null, e => {
  console.log(e); // "test"
});
```

### `.allRejected()` <Badge variant="static" /><Badge variant="method" />

`.allRejected` is a function that collects only the reasons why `StatefulPromise` objects are rejected.

```ts
allRejected(promises: Iterable<unknown>): StatefulPromise<unknown[]>;
allRejected<T>(
  promises: Iterable<T>,
  extract: (item: T) => unknown,
): StatefulPromise<unknown[]>;
```

#### Arguments

##### `promises`

An iterable object. It can contain values other than `StatefulPromise` objects, but they will be treated as errors. Regular `Promise` objects will also be ignored.

##### `extract`

A function that extracts `StatefulPromise` objects from `promises`.

#### Return Value

It returns a `StatefulPromise` object resolved with an array of reasons why `StatefulPromise` objects were rejected.
