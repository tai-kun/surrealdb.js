`mutex`는 하나 이상의 클래스 메소드를 동시에 실행할 수 있는 횟수를 1로 제한하는 비동기 처리를 위한 데코레이터입니다.

<Aside type="caution">
  빌드된 SDK를 JavaScript에서 사용하는 경우 문제가 없지만, SDK 소스 코드(TypeScript)를 직접 사용하는 경우 Stage 3 데코레이터 지원이 필요합니다.
</Aside>

### 임포트

```ts
import { mutex } from "@tai-kun/surrealdb/utils";
```

### 사용 방법

```ts
function mutex(
  target: (...args: any) => PromiseLike<any>,
  context: ClassMethodDecoratorContext,
): (this: any, ...args: any) => Promise<any>;
```

#### 인자

##### `target`

클래스 메소드입니다.

##### `context`

Stage 3 클래스 메소드용 데코레이터 컨텍스트입니다.

#### 리턴 값

동시 실행 횟수가 1로 제한된 비동기 함수입니다.

#### 예

다음 예제는 `mutex`를 사용했을 때와 사용하지 않았을 때 비동기 처리의 실행 순서가 어떻게 달라지는지 보여줍니다.

```ts
import { mutex } from "@tai-kun/surrealdb/utils";

class Runner {
  async runWithoutMutex(ms: number, value: string) {
    await sleep(ms);
    console.log(value);
  }

  @mutex
  async runWithMutex(ms: number, value: string) {
    await sleep(ms);
    console.log(value);
  }
}

const runner = new Runner();
// mutex 사용하지 않음
await Promise.all([
  runner.runWithoutMutex(1000, "A"),
  runner.runWithoutMutex(500, "B"),
  runner.runWithoutMutex(0, "C"),
]);
// mutex 사용
await Promise.all([
  runner.runWithMutex(1000, "A"),
  runner.runWithMutex(500, "B"),
  runner.runWithMutex(0, "C"),
]);
```

출력:

```js
C
B
A
A
B
C
```
