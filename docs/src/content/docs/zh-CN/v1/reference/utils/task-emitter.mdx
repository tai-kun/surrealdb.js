`TaskEmitter` 是一个用于 `TaskQueue` 管理的异步任务的事件发射器类。

### 导入

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";
```

### `.constructor()`

```ts
new TaskEmitter<T>();
```

#### 参数

##### `T`

一个包含事件名称和事件参数（数组）类型的键值对。

#### 返回值

当通过 `new` 调用时，`TaskEmitter` 会返回它的实例。

### `.on()` <Badge variant="instance" /><Badge variant="method" />

`.on` 是一个将事件监听器注册到事件发射器上的函数。如果尝试多次将同一个函数注册到同一个事件，除了第一次之外，其余注册操作都会被忽略。

```ts
on(event: string | number, listener: TaskListener): void;
```

#### 参数

##### `event`

要注册事件监听器的事件名称。

##### `listener`

要注册的事件监听器。该函数将接收与 `TaskQueue` 的任务运行器相同的参数，之后还会接收通过 `.emit` 传递的可变长度参数。

#### 返回值

无。

#### 例子

以下示例会捕捉 "log" 事件，并将传递给发射器的值输出到标准输出：

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";

type Events = {
  log: string[];
};

const emitter = new TaskEmitter<Events>();

emitter.on("log", (_, ...args) => {
  console.log(...args);
});

emitter.emit("log", "Hello");
emitter.emit("log", "World", "!!!");

await emitter.idle();
```

输出：

```js
Hello
World !!!
```

### `.off()` <Badge variant="instance" /><Badge variant="method" />

`.off` 是一个从事件发射器中注销事件监听器的函数。如果省略事件监听器，则会注销目标事件名称上注册的所有事件监听器。

```ts
off(event: string | number, listener?: TaskListener): void;
```

#### 参数

##### `event`

要注销事件监听器的事件名称。

##### `listener`

要注销的事件监听器。如果省略该参数，则会注销目标 `event` 上注册的所有事件监听器。

#### 返回值

无。

### `.once()` <Badge variant="instance" /><Badge variant="method" />

`.once` 与 `.on` 类似，但它只捕捉一次事件。它返回的是 `StatefulPromise`，而不是回调函数，可以通过它接收返回值。

```ts
once(
  event: string | Number,
  options?: TaskListenerOptions,
): StatefulPromise<unknown[]>;
```

#### 例子

以下示例会捕捉一次 "log" 事件，并将传递给发射器的值输出到标准输出：

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";

type Events = {
  log: string[];
};

const emitter = new TaskEmitter<Events>();

const promise = emitter.once("log");

emitter.emit("log", "Hello");
emitter.emit("log", "World", "!!!");

const recived = await promise;
console.log(recived);

await emitter.idle();
```

输出：

```js
[ 'Hello' ]
```

#### 参数

##### `event`

要等待的事件名称。

##### `options`

事件监听器的选项。可以传递中止信号。

#### 返回值

返回一个以 `.emit` 的参数（数组）解析的 `StatefulPromise`。

### `.emit()` <Badge variant="instance" /><Badge variant="method" />

`.emit` 是一个向事件监听器发送值的函数。

```ts
emit(
  event: string | number,
  ...args: unknown[]
): undefined | StatefulPromise<unknown>[];
```

#### 参数

##### `event`

要发送到的事件名称。

##### `args`

要发送给目标事件监听器的值。

#### 返回值

如果没有事件监听器，则返回 `undefined`。如果有事件监听器，则返回一个 `StatefulPromise` 数组，这些 `StatefulPromise` 会一直等待它们被解析或拒绝。通常情况下，不需要处理这些 `StatefulPromise`。只有在需要在当前上下文等待由事件触发的副作用、或对事件监听器的解析或拒绝感兴趣时，才需要使用它们。

### `.idle()` <Badge variant="instance" /><Badge variant="method" />

`.idle` 是一个等待所有当前运行的事件监听器完成的函数。

```ts
idle(): StatefulPromise<void>;
```

#### 参数

无。

#### 返回值

返回一个以 `void` 解析的 `StatefulPromise`。

### `.abort()` <Badge variant="instance" /><Badge variant="method" />

`.abort` 是一个向所有事件监听器的 `signal` 发送中止事件的函数。

```ts
abort(reason?: unknown): void;
```

#### 参数

##### `reason`

中止的原因。

#### 返回值

无。
