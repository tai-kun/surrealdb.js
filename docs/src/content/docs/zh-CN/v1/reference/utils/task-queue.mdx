`TaskQueue` 是一个用于管理异步任务的类。它不提供任何控制并发执行的方法。所有的任务运行器都异步执行。

### 导入

```ts
import { TaskQueue } from "@tai-kun/surrealdb/utils";
```

### `.constructor()`

```ts
new TaskQueue();
```

#### 参数

无。

#### 返回值

当通过 `new` 调用时，`TaskQueue` 返回其实例。

### `.count` <Badge variant="instance" /><Badge variant="property" /><Badge variant="readonly" />

表示当前添加到队列中的任务运行器数量。

### `.add()` <Badge variant="instance" /><Badge variant="method" />

`.add` 将任务运行器添加到队列中。

```ts
add<T>(
  runner: (args: TaskRunnerArgs) => T | PromiseLike<T>,
  options?: TaskOptions,
): StatefulPromise<T>;
```

<Aside type="caution">
  如果任务运行器的执行时间可能很长，建议在任务运行器接收到中止事件时，通过传递给它的中止信号立即中止任务。
</Aside>

#### 参数

##### `runner`

任务运行器。任务运行器的参数包含一个包含中止信号 `signal` 的 `TaskRunnerArgs`。

##### `options`

任务运行器的选项。可以传递此任务运行器的中止信号。

#### 返回值

返回一个用任务运行器的返回值解析的 `StatefulPromise`。

#### 示例

```ts
import { TaskQueue } from "@tai-kun/surrealdb/utils";

const queue = new TaskQueue();
const promise = queue.add(async ({ signal }) => {
  const response = await fetch("https://example.com/", { signal });
  return await response.text();
});

const text = await promise;
console.log(text);
```

### `.idle()` <Badge variant="instance" /><Badge variant="method" />

`.idle` 是一个用于等待队列为空的函数。队列中删除任务运行器是在任务运行器完成（无论解析还是拒绝）之后立即进行的。因此，队列为空意味着所有任务运行器都已完成执行。

```ts
idle(): StatefulPromise<void>;
```

<Aside type="caution">
  `.idle` 不会捕获 `.add` 方法中添加的任务运行器抛出的错误。由于任务运行器在 `StatefulPromise` 内执行，因此无法在没有错误处理的情况下捕获任务运行器的错误。
</Aside>

#### 参数

无。

#### 返回值

返回一个用 `void` 解析的 `StatefulPromise`。

#### 示例

以下示例等待队列中添加的任务运行器完成（无论解析还是拒绝）：

```ts
import { TaskQueue } from "@tai-kun/surrealdb/utils";

const queue = new TaskQueue();
const results: string[] = [];
queue.add(async () => {
  results.push("Hello");
});
queue.add(() => {
  throw null;
});

console.log(queue.count);

await queue.idle();

console.log(queue.count);
console.log(results);
```

输出：

```js
2
0
[ 'Hello' ]
```

在上面的示例中，`throw null` 没有被任何地方捕获，脚本正常退出。要捕获任务运行器的错误，需要单独进行错误处理。

### `.abort()` <Badge variant="instance" /><Badge variant="method" />

`.abort` 是一个函数，用于向所有任务运行器的 `signal` 发送中止事件。

```ts
abort(reason?: unknown): void;
```

#### 参数

##### `reason`

中止的原因。

#### 返回值

无。
