`TaskEmitter` 是一个为 `TaskQueue` 管理的异步任务提供事件发射器功能的类。

### 导入

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";
```

### `.constructor()`

```ts
new TaskEmitter<T>();
```

#### 参数

##### `T`

一个用 key-value 形式表示事件名及其对应参数类型（数组）的类型。

#### 返回值

当通过 `new` 调用时，`TaskEmitter` 会返回其实例。

### `.on()` <Badge variant="instance" /><Badge variant="method" />

`.on` 函数用于向事件发射器注册事件监听器。如果试图多次注册同一个事件的同一个监听器，除了第一次注册之外，其他注册都会被忽略。

```ts
on(event: string | number, listener: TaskListener): void;
```

#### 参数

##### `event`

要注册监听器的事件名称。

##### `listener`

要注册的事件监听器。该函数将接收与 `TaskQueue` 的任务运行器相同的参数，以及 `.emit` 传递的参数（可变数量）。

#### 返回值

无。

#### 示例

以下示例演示如何捕捉 "log" 事件并将发射器传递的值输出到标准输出：

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";

type Events = {
  log: string[];
};

const emitter = new TaskEmitter<Events>();

emitter.on("log", (_, ...args) => {
  console.log(...args);
});

emitter.emit("log", "Hello");
emitter.emit("log", "World", "!!!");

await emitter.idle();
```

输出：

```js
Hello
World !!!
```

### `.off()` <Badge variant="instance" /><Badge variant="method" />

`.off` 函数用于从事件发射器中注销事件监听器。如果省略监听器，则会注销所有与目标事件名称相关联的事件监听器。

```ts
off(event: string | number, listener?: TaskListener): void;
```

#### 参数

##### `event`

要注销监听器的事件名称。

##### `listener`

要注销的事件监听器。如果省略，则会注销所有与目标 `event` 相关联的事件监听器。

#### 返回值

无。

### `.once()` <Badge variant="instance" /><Badge variant="method" />

`.once` 函数类似于 `.on`，但它只捕捉一次事件。此外，它不会返回回调函数，而是通过返回值 `StatefulPromise` 接收值。

```ts
once(
  event: string | Number,
  options?: TaskListenerOptions,
): StatefulPromise<unknown[]>;
```

#### 示例

以下示例演示如何只捕捉一次 "log" 事件并将发射器传递的值输出到标准输出：

```ts
import { TaskEmitter } from "@tai-kun/surrealdb/utils";

type Events = {
  log: string[];
};

const emitter = new TaskEmitter<Events>();

const promise = emitter.once("log");

emitter.emit("log", "Hello");
emitter.emit("log", "World", "!!!");

const recived = await promise;
console.log(recived);

await emitter.idle();
```

输出：

```js
[ 'Hello' ]
```

#### 参数

##### `event`

要等待的事件名称。

##### `options`

事件监听器的选项。可以传递中止信号。

#### 返回值

返回一个用 `.emit` 的参数（数组）来解析的 `StatefulPromise`。

### `.emit()` <Badge variant="instance" /><Badge variant="method" />

`.emit` 函数用于向事件监听器发送值。

```ts
emit(
  event: string | number,
  ...args: unknown[]
): undefined | StatefulPromise<unknown>[];
```

#### 参数

##### `event`

要发送的目标事件名称。

##### `args`

发送给目标事件监听器的值。

#### 返回值

如果事件监听器不存在，则返回 `undefined`。如果事件监听器存在，则返回一个 `StatefulPromise` 数组，这些 `StatefulPromise` 将等待监听器解析或拒绝。通常情况下，不需要处理这些 `StatefulPromise`。如果需要在当前上下文中等待事件触发的副作用，或者需要关注事件监听器解析或拒绝，那么可以使用这些 `StatefulPromise`。

### `.idle()` <Badge variant="instance" /><Badge variant="method" />

`.idle` 函数用于等待所有正在运行的事件监听器完成。

```ts
idle(): StatefulPromise<void>;
```

#### 参数

无。

#### 返回值

返回一个解析为 `void` 的 `StatefulPromise`。

### `.abort()` <Badge variant="instance" /><Badge variant="method" />

`.abort` 函数用于向所有事件监听器的 `signal` 发送中止事件。

```ts
abort(reason?: unknown): void;
```

#### 参数

##### `reason`

中止的原因。

#### 返回值

无。
