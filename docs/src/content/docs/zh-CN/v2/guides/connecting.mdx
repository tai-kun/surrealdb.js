### 导入

```ts
import { Surreal } from "@tai-kun/surrealdb";
```

### `.constructor()`

```ts
new Surreal();
```

#### 参数

无。

#### 返回值

通过 `new` 调用时，`Surreal` 返回其实例。

### `.connect()` <Badge variant="instance" /><Badge variant="method" />

连接 SurrealDB 终结点。该方法是异步执行的，但在同一个实例中，不会与其他 `.connect()` 或 `.close()` 重复执行。

```ts
connect(
  endpoint: string | URL,
  options?: ClientConnectOptions,
): Promise<void>;
```

#### 参数

##### `endpoint`

以 URL 形式指定 SurrealDB 终结点。如果 URL 路径的末尾不是 `/rpc`，则会自动添加。因此，如果传递 `http://localhost:8000` 作为参数，则将连接到 `http://localhost:8000/rpc`。

##### `options`

连接时的选项。

`signal?: AbortSignal`
:   用于中止连接的中止信号。默认情况下，将设置一个在 15 秒后超时的中止信号。

#### 返回值

返回一个以 `undefined` 解决的 `Promise` 对象。

#### 例子

以下示例使用 WebSocket 协议连接到 SurrealDB：

```ts
import { Surreal } from "@tai-kun/surrealdb";

const db = new Surreal();

await db.connect("ws://localhost:8000");
```

不能在没有 `.close()` 断开的情况下连接到不同的终结点。注意，与先前建立连接的终结点的连接将保持：

```ts
import { Surreal } from "@tai-kun/surrealdb";

const db = new Surreal();

await db.connect("ws://localhost:8000");
await db.connect("ws://localhost:1129"); // throws ConnectionConflictError: Connection conflict between ws://localhost:8000/rpc and ws://localhost:1129/rpc.
```

`.connect()` 不会在异步处理的上下文中同时执行，因此以下示例也会导致连接失败。通常不知道哪个终结点会先建立连接，因此错误消息可能会因情况而异：

```ts
import { Surreal } from "@tai-kun/surrealdb";

const db = new Surreal();

await Promise.all([
  db.connect("ws://localhost:8000"),
  db.connect("ws://localhost:1129"),
]); // throws ConnectionConflictError
```

### `.close()` <Badge variant="instance" /><Badge variant="method" />

断开与 SurrealDB 的连接。如果尚未连接或已断开连接，则不会抛出错误。该方法是异步执行的，但在同一个实例中，不会与 `.connect()` 或其他 `.close()` 重复执行。

```ts
close(options?: ClientCloseOptions): Promise<void>
```

#### 参数

##### `options`

断开连接时的选项。

`force?: boolean`
:   如果设置为 `true`，则会将中止信号发送到传递给 `Surreal` 对象的事件监听器的中止信号。预期事件监听器会立即中断，即使处理不完整。

`signal?: AbortSignal`
:   用于中止连接的中止信号。默认情况下，将设置一个在 15 秒后超时的中止信号。

#### 返回值

返回一个以 `undefined` 解决的 `Promise` 对象。

#### 例子

以下示例使用 WebSocket 协议连接到 SurrealDB，然后断开连接：

```ts
import { Surreal } from "@tai-kun/surrealdb";

const db = new Surreal();

await db.connect("ws://localhost:8000");
await db.close();
```

### `.getConnectionInfo()` <Badge variant="instance" /><Badge variant="method" />

获取连接信息。连接信息始终包含内部信息的复制值。因此，连接信息将包含 `URL` 对象，但直接修改它不会改变 `Surreal` 对象内部的终结点信息。如果尚未请求连接，则为 `undefined`。

```ts
getConnectionInfo(): ConnectionInfo | undefined;
```

#### 参数

无。

#### 返回值

如果建立了连接，则返回连接信息。否则，返回 `undefined`。连接信息包含以下项目：

`state: "connecting" | "open" | "closing" | "closed"`
:   表示当前连接状态的数值。每个数值表示以下含义：

`endpoint: URL | null`
:   连接到的终结点。仅在断开连接状态下为 `null`。

`namespace: string | null`
:   当前选择的命名空间。如果没有选择命名空间，则为 `null`。

`database: string | null`
:   当前选择的数据库名称。如果没有选择数据库，则为 `null`。

`token: string | null`
:   当前已认证的令牌。如果没有注册或登录，则为 `null`。

在每个连接状态下，连接信息可以取的值如以下表格所示：

| state          | endpoint | namespace      | database       | token          |
| -------------- | -------- | -------------- | -------------- | -------------- |
| `"connecting"` | `URL`    | `null`         | `null`         | `null`         |
| `"open"`       | `URL`    | `string\|null` | `string\|null` | `string\|null` |
| `"closing"`    | `URL`    | `string\|null` | `string\|null` | `string\|null` |
| `"closed"`     | `null`   | `null`         | `null`         | `null`         |

### `.state` <Badge variant="instance" /><Badge variant="property" /><Badge variant="readonly" />

表示当前连接状态的数值。

```ts
state: "connecting" | "open" | "closing" | "closed"
```

为了方便起见，即使未建立连接，也会为 `"closed"`（不是 `undefined`）。

相关：请参阅 [`.getConnectionInfo()`](#getconnectioninfo)。

### `.endpoint` <Badge variant="instance" /><Badge variant="property" /><Badge variant="readonly" />

连接到的终结点。仅在断开连接状态下为 `null`。如果未建立连接，则为 `undefined`。

```ts
endpoint: string | null | undefined
```

相关：请参阅 [`.getConnectionInfo()`](#getconnectioninfo)。

### `.namespace` <Badge variant="instance" /><Badge variant="property" />

当前选择的命名空间。如果没有选择命名空间，则为 `null`。如果未建立连接，则为 `undefined`。

```ts
namespace: string | null | undefined
```

相关：请参阅 [`.getConnectionInfo()`](#getconnectioninfo)。

### `.database` <Badge variant="instance" /><Badge variant="property" />

当前选择的数据库名称。如果没有选择数据库，则为 `null`。如果未建立连接，则为 `undefined`。

```ts
database: string | null | undefined
```

相关：请参阅 [`.getConnectionInfo()`](#getconnectioninfo)。

### `.token` <Badge variant="instance" /><Badge variant="property" />

当前已认证的令牌。如果没有注册或登录，则为 `null`。如果未建立连接，则为 `undefined`。

```ts
token: string | null | undefined
```

相关：请参阅 [`.getConnectionInfo()`](#getconnectioninfo)。

### `.on()` <Badge variant="instance" /><Badge variant="method" />

注册事件监听器。

```ts
off(
  event:
    | "connecting" | "open" | "closing" | "closed"
    | `rpc_${RpcMethod}_${number}`
    | `live_${string}`
    | "error",
  listener: (taskRunnerArgs, ...eventArgs) => void | PromiseLike<void>,
): void
```

#### 参数

##### `event`

要监视的事件名称。

`"connecting" | "open" | "closing" | "closed"`
:   当连接信息发生转换或转换失败时发生的事件。

`rpc_${RpcMethod}_${number}`
:   双向通信中发生 RPC 响应时发生的事件。当前仅在 WebSocket 协议中使用，在 HTTP 协议中不使用此事件。`RpcMethod` 是例如 `ping` 或 `use` 等 PRC 方法名称。`number` 是双向通信中用于将请求/响应关联起来的标识符，它可以取 1 到 2^53-1 之间的整数。

`live_${string}`
:   接收实时查询的事件时发生的事件。当前仅在 WebSocket 协议中使用，在 HTTP 协议中不使用此事件。`string` 是实时查询的 UUID。

`error`
:   错误事件。当前仅在 WebSocket 协议中使用，在 HTTP 协议中不使用此事件。

##### `listener`

事件发生时执行的函数。请参阅 [任务发射器](/v2/api/utils/task-emitter#on)。传递给 `eventArgs` 的值取决于事件。

`"connecting" | "open" | "closing" | "closed"`
:   以下之一：
:   - `{ state: "connecting" | "open" | "closing" | "closed" }`
    - `{ state: "connecting" | "open" | "closing" | "closed", error: unknown }`

`rpc_${RpcMethod}_${number}`
:   以下之一：
:   - `{ id: string, result: unknown }`
    - `{ id: string, error: { code: number, message: string } }`

`live_${string}`
:   通常模式的实时查询：
:   - `{ action: "CREATE" | "UPDATE" | "DELETE", result: object }`
:   差异获取模式的实时查询：
:   - `{ action: "CREATE" | "UPDATE", result: object[] }`
    - `{ action: "DELETE", result: object }`

`error`
:   以下之一：
:   - [`HttpEngineError`](/v2/api/errors/#httpengineerror)
    - [`WebSocketEngineError`](/v2/api/errors/#websocketengineerror)

#### 返回值

无。

### `.once()` <Badge variant="instance" /><Badge variant="method" />

与 `.on()` 相似，但只捕获一次事件。

```ts
once(
  event:
    | "connecting" | "open" | "closing" | "closed"
    | `rpc_${RpcMethod}_${number}`
    | `live_${string}`
    | "error",
  options?: TaskListenerOptions,
): StatefulPromise<unknown[]>
```

#### 参数

##### `event`

与 `.on()` 中的 `event` 相同。

##### `options`

任务监听器的选项。请参阅 [任务发射器](/v2/api/utils/task-emitter#once)。

#### 返回值

返回一个以 `.on()` 的任务监听器的 `eventArgs` 解决的 `StatefulPromise`。

### `.off()` <Badge variant="instance" /><Badge variant="method" />

取消 `.on()` 中注册的事件监听器。

```ts
off(
  event:
    | "connecting" | "open" | "closing" | "closed"
    | `rpc_${RpcMethod}_${number}`
    | `live_${string}`
    | "error",
  listener: (taskRunnerArgs, ...eventArgs) => void | PromiseLike<void>,
): void
```

#### 参数

##### `event`

要取消注册事件监听器的事件名称。

##### `listener`

要取消注册的事件监听器。

#### 返回值

无。
